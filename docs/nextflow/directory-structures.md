# Directory Structures

## `data/valid` Directory Structure

This directory contains all input data used by the Nextflow pipeline.

```
data/valid/
├── assembled_genome.fasta     # if the provided sequence is NOT too fragmented
├── reference_genome.fasta
├── mod_config_[0..4].fasta    # Presence of multiple contings in case of a fragmented assembly
├── ref_plasmid.fa             # Reference plasmid sequences (if identified within a validation step)
├── mod_plasmid.fa             # Modified/assembled plasmid sequences (if identified within a validation step)
├── ref_feature.gff            # Genome annotation file GTF/GFF (if identified within a validation step)
├── validated_params.json      # Validated paths and flags for Nextflow (-params-file)
│
├── valiation.log
├── report.txt
│
├── illumina/                  
│   ├── SampleName_1.fastq.gz  
│   ├── SampleName_2.fastq.gz  
│
├── ont/                       
│   └── SampleName.fastq.gz
│
└── pacbio/                   
    └── SampleName.fastq.gz

```

### File Descriptions

| File / Folder              | Description                                                                   |
| -------------------------- | ----------------------------------------------------------------------------- |
| `reference_genome.fasta`   | The primary reference genome sequence.                                        |
| `assembled_genome.fasta`   | Assembled or modified genome for comparison/analysis.                         |
| `ref_plasmid.fa`           | Reference plasmid sequences.                                                  |
| `mod_plasmid.fa`           | Modified or assembled plasmid sequences.                                      |
| `ref_feature.gff`          | GFF feature file for annotations.                                             |
| `validated_params.json`    | JSON file produced by the validation step; loaded by Nextflow via `-params-file`. Contains validated FASTA paths and pipeline flags (e.g. `run_ref_x_mod`). |
| `valiation.log`            | A log file generated by a validation step.                                    |
| `report.txt`               | Statistics on the input sequences generated by a validation step.             |
| `illumina/`                | Paired-end Illumina short reads.                                              |
| `ont/`                     | Oxford Nanopore long reads.                                                   |
| `pacbio/`                  | PacBio long reads.                                                            |

## `data/outputs` Directory Structure

After successful pipeline execution, the outputs are organized as follows:

```
data/outputs
├── fasta_ref_mod       → Results from reference vs modified FASTA comparison (if run_ref_x_mod set to true in `data/validation/validated_params.json`)
├── illumina            → Short-read (Illumina) mapping results
├── logs                → Pipeline logs and Nextflow reports
├── ont                 → Long-read (Oxford Nanopore) mapping results
├── pacbio              → Long-read (PacBio) mapping results
├── tables              → Per-SV csv tables
├── truvari             → Variant comparison results from Truvari (if run_truvari set to true in `data/validation/validated_params.json`)
└── unmapped_stats      → Summary statistics of unmapped reads for each workflow
```
